# Security Policy

## Status

WeJoy v0.1 is suitable for controlled mock-payment testing only. It has not received a production security audit and must not process real customer funds.

## Reporting

Report vulnerabilities through the repository's private GitHub security advisory feature. Do not open a public issue for an unpatched authentication, payment, privacy, signing-key, or remote-code-execution problem.

Include affected commit, reproduction steps, impact, and a proposed mitigation when available. Do not include real personal data or credentials.

## Supported Version

Only the latest commit on `main` is supported during the MVP phase.
